## Project Overview

### Problem Statement
Modern DevOps environments require structured incident handling, automated deployment pipelines, and monitoring mechanisms. This project simulates the backend lifecycle of incident management in a DevOps environment, focusing on containerization, CI/CD automation, monitoring workflows, and frontend interaction.

### Objectives
- Implement incident lifecycle simulation (Create, View, Resolve)
- Containerize the application using Docker
- Implement CI/CD pipeline using GitHub Actions
- Add automated unit testing
- Add Selenium UI testing structure
- Simulate monitoring setup with health endpoint

---

## Key Features

- Create incidents with severity
- View all incidents
- Resolve incidents
- Health check endpoint (`/health`)
- Web-based dashboard UI
- Dockerized deployment
- Git branching workflow
- CI pipeline integration
- Selenium UI test structure
