## Project Overview

### Problem Statement
In modern DevOps environments, production systems frequently encounter incidents such as service crashes, high resource usage, and deployment failures. This project simulates a DevOps Incident Management System where incidents can be created, monitored, and resolved through a containerized application integrated with CI/CD pipelines.

### Objectives
- [x] Implement a simple incident management backend
- [x] Containerize the application using Docker
- [x] Implement CI/CD using GitHub Actions
- [x] Add automated testing
- [x] Simulate monitoring structure

### Key Features
- Create incidents
- View all incidents
- Resolve incidents
- Dockerized deployment
- Automated CI pipeline
- Unit testing support
